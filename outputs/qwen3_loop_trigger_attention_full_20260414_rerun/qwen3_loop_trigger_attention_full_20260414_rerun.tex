\section*{Object}

This note compares the two query positions that matter for the current question on the same saved March 22--23 Qwen3-1.7B instruct prompt-profile rows:

\begin{itemize}
  \item \texttt{trigger\_end}: the final token of the twentieth repeated $30$-gram
  \item \texttt{trigger\_start}: the token immediately before that twentieth repeated copy begins
\end{itemize}

This report uses \texttt{trigger\_start} only for the token right before the final repeated copy begins.

Everything else is held fixed:

\begin{itemize}
  \item model: \texttt{Qwen/Qwen3-1.7B}
  \item loop definition: \texttt{n=30}, \texttt{k=20}
  \item selection rule: recomputed trigger prefix must exactly match saved \texttt{first\_loop\_prefix\_length}
  \item selected rows: \texttt{811 / 820} replayable loop rows
  \item dataset counts: \texttt{GPQA 121}, \texttt{AIME 30}, \texttt{MATH-500 68}, \texttt{MMLU-Pro 130}, \texttt{LiveCodeBench 462}
  \item total prefix lengths: median \texttt{9846}, \texttt{p95 = 22321}, max \texttt{28830}
\end{itemize}

There is still no extra mid-prefix truncation inside any run. Each forward pass sees the full prompt plus the completion prefix through the saved first loop trigger.

\section*{Measurement Definition}

For each selected row:

\begin{enumerate}
  \item Recompute the loop trigger with \texttt{find\_ngram\_loop\_trigger(..., n=30, k=20)}.
  \item Keep only rows whose recomputed \texttt{first\_loop\_prefix} matches the saved \texttt{first\_loop\_prefix\_length}.
  \item Replay the model on \texttt{prompt\_token\_ids + completion\_token\_ids[:first\_loop\_prefix\_length]}.
  \item Apply an explicit causal mask inside the probe wrapper so keys with \texttt{key\_position > query\_position} are forced to zero.
\end{enumerate}

The two reported query positions are:

\begin{itemize}
  \item \texttt{trigger\_end = prompt\_len + trigger\_end}
  \item \texttt{trigger\_start = prompt\_len + trigger\_start - 1}
\end{itemize}

Key positions are partitioned into disjoint bins:

\begin{itemize}
  \item \texttt{prompt}
  \item \texttt{previous\_loop}
  \item \texttt{current\_trigger}
  \item \texttt{recent\_nonloop}
  \item \texttt{other\_completion}
\end{itemize}

\texttt{last\_previous\_loop} is still tracked as a subset of \texttt{previous\_loop}, but it is not plotted as a separate mass line here.

There is no separate self bin. The query token contributes to the region it belongs to. At \texttt{trigger\_start}, \texttt{current\_trigger} is exactly zero because the full trigger span is in the future.

Summary statistics are:

\begin{itemize}
  \item \texttt{mean\_*\_mass}: summed attention mass on that bin, averaged across heads, then across selected rows
  \item \texttt{top1\_fraction\_*}: fraction of heads whose argmax lands in that bin
\end{itemize}

\texttt{attention\_mass\_by\_layer\_comparison.pdf} plots row-weighted overall bin mass from layer $0$ to layer $27$ for the two reported query positions. The plotted lines are:

\begin{itemize}
  \item \texttt{trigger\_end}: \texttt{prompt}, \texttt{previous\_loop}, \texttt{current\_trigger}, and \texttt{other\_completion}
  \item \texttt{trigger\_start}: \texttt{prompt}, \texttt{previous\_loop}, and \texttt{other\_completion}
\end{itemize}

\texttt{recent\_nonloop} is still measured in the raw summaries and final-layer table, but it is intentionally omitted from the plot.

Main read from the curves:

\begin{itemize}
  \item \texttt{trigger\_end}: previous-loop mass rises early, peaks at layer $6$ (\texttt{0.193}), and then falls as the final layer becomes prompt-dominant; by layer $27$, \texttt{current\_trigger} (\texttt{0.176}) is still larger than \texttt{other\_completion} (\texttt{0.116}).
  \item \texttt{trigger\_start}: \texttt{current\_trigger} is identically \texttt{0}, previous-loop mass peaks later at layer $16$ (\texttt{0.211}), and the strongest competition there is between \texttt{prompt} (\texttt{0.260}) and \texttt{other\_completion} (\texttt{0.329}).
\end{itemize}

So the two views differ materially. Moving the query one token earlier removes \texttt{current\_trigger}, raises final-layer previous-loop mass, and shifts more of the non-prompt mass into \texttt{other\_completion}.

\section*{Conclusion}

The read is now:

\begin{itemize}
  \item the pre-trigger object does show nontrivial attention to previous loops:
    final-layer previous-loop mass \texttt{0.069}, final-layer top-1 previous-loop \texttt{0.027}, and overall previous-loop peak at layer $16$ with mass \texttt{0.211};
  \item but the final layer is still mostly prompt-focused rather than previous-loop-focused:
    final-layer prompt mass \texttt{0.634} and final-layer top-1 prompt \texttt{0.872}.
\end{itemize}

So the strongest honest statement is not ``Qwen3 is not paying attention to previous loops.'' The stronger-supported statement is:

\begin{quote}
At the token immediately before the twentieth repeated copy begins, the final layer is still mostly prompt-focused, while previous-loop evidence is present and peaks in the middle of the stack.
\end{quote}
