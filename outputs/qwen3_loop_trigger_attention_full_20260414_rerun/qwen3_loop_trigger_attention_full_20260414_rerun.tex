\section*{Object}

This note answers the current loop-trigger question on the saved March 22--23 Qwen3-1.7B instruct prompt-profile archives: when Qwen3 reaches the repeated $30$-gram that trips the $n=30$, $k=20$ loop detector, what is the query state attending to?

This is the full rerun, not the earlier $14$-row bounded pilot. The analyzed object is:

\begin{itemize}
  \item model: \texttt{Qwen/Qwen3-1.7B}
  \item loop definition: \texttt{n=30}, \texttt{k=20}
  \item selection rule: all rows whose reconstructed trigger prefix matches the saved \texttt{first\_loop\_prefix\_length}, with \texttt{max\_trigger\_prefix=40960} and \texttt{max\_samples\_per\_dataset=1000}
  \item selected rows: \texttt{811 / 820} loop rows
  \item dataset counts: \texttt{GPQA 121}, \texttt{AIME 30}, \texttt{MATH-500 68}, \texttt{MMLU-Pro 130}, \texttt{LiveCodeBench 462}
  \item total prefix lengths on the selected rows: median \texttt{9846}, \texttt{p95 = 22321}, max \texttt{28830}
\end{itemize}

There is no extra mid-prefix truncation inside this rerun. Each HF forward sees the full prompt plus the completion prefix through the saved first loop trigger.

\section*{How The Measurement Is Defined}

\begin{enumerate}
  \item Recompute the loop trigger on the reconstructed completion token IDs with \texttt{find\_ngram\_loop\_trigger(..., n=30, k=20)}.
  \item Keep only rows where the recomputed \texttt{first\_loop\_prefix} exactly matches the saved \texttt{first\_loop\_prefix\_length}.
  \item Replay the model on \texttt{prompt\_token\_ids + completion\_token\_ids[:first\_loop\_prefix\_length]}.
  \item Define the query token as \texttt{query\_position = prompt\_len + trigger\_end = total\_prefix\_length - 1}. This is the final token of the twentieth repeated $30$-gram, not \texttt{trigger\_start - 1}.
  \item For every layer and head, capture the softmax attention from that one query token while preserving the model's masking behavior.
  \item Partition key positions into disjoint bins:
  \begin{itemize}
    \item \texttt{prompt}: all prompt positions
    \item \texttt{current\_trigger}: the current triggering $30$-gram, including self
    \item \texttt{previous\_loop}: all earlier copies of that same $30$-gram, excluding overlap with the current trigger span
    \item \texttt{last\_previous\_loop}: the immediately preceding loop copy, again excluding overlap
    \item \texttt{recent\_nonloop}: the previous $256$ completion positions that are in neither \texttt{previous\_loop} nor \texttt{current\_trigger}
    \item \texttt{other\_completion}: any remaining completion positions outside those bins
  \end{itemize}
  \item Summaries are defined as:
  \begin{itemize}
    \item \texttt{mean\_*\_mass}: for one row, sum the attention weights on that bin, average across heads, then average those row-level values across the selected rows
    \item \texttt{top1\_fraction\_*}: fraction of heads whose argmax lands in that bin
  \end{itemize}
\end{enumerate}

So the measured object is the next-token state after the full triggering copy has already been written. That makes this a defensible trigger-state read, but it is not yet the stricter ``about to enter the loop'' object.

\section*{Conclusion}

For the exact surface measured here:

\begin{itemize}
  \item the strong claim ``Qwen3 is not paying attention to previous loops'' is too strong for the whole stack;
  \item the narrower claim ``the final-layer trigger state is mostly prompt-focused, not previous-loop-focused'' is supported by the full rerun;
  \item the current result does not yet answer the stricter ``about to enter the loop'' question, because the query token is the end of the triggering copy, not its start.
\end{itemize}

If we want the more literal pre-entry object next, the rerun should move the query to \texttt{trigger\_start - 1} or \texttt{trigger\_start}, ideally with matched non-loop controls.

\section*{Future Rollouts}

The saver-side fixes are already on this branch:

\begin{itemize}
  \item prompt-profile archives now save exact \texttt{completion\_token\_ids} plus structured \texttt{loop\_trigger}
  \item the rollout-stats collector now writes \texttt{\_\_rollout\_archive.jsonl.gz} with prompt/completion token IDs and trigger metadata
\end{itemize}

So this exact reconstruction gap should not recur on future rollouts.
