\section*{Question}
What does the first locked full-train pass say about the current two-head prompt-profile surface under the fixed saved contract?

\begin{itemize}[leftmargin=1.5em]
\item regression head: \texttt{mean\_relative\_length}
\item binary head: \texttt{majority\_s\_0.5}
\item feature surface: prompt-prefill activations only
\item view comparison: layerwise \texttt{ensemble} versus \texttt{last\_layer}
\end{itemize}

\section*{Setup}
\begin{itemize}[leftmargin=1.5em]
\item Runtime object: Slurm job \texttt{2043} on \texttt{2} GPUs, completed at \texttt{2026-04-04 00:10 UTC}.
\item Data object: reused the saved March prompt-profile archives; no rerollout.
\item Selector: report the standardized \texttt{best\_loss} ledger and keep \texttt{best\_rank} diagnostic-only.
\item Mechanism split:
  \begin{itemize}[leftmargin=1.5em]
  \item for regression \texttt{mean\_relative\_length}, \texttt{ensemble} means one MLP per layer with \texttt{mean\_prob} aggregation;
  \item for binary \texttt{majority\_s\_0.5}, \texttt{ensemble} means one MLP per layer with \texttt{vote\_fraction} aggregation over the per-layer 0/1 predictions.
  \end{itemize}
\item Metadata baselines below are train-fit held-out scorers, not raw correlations.
\item \texttt{effective\_budget} is constant at \texttt{30000} on this run, so the joint regression control collapses to prompt length.
\end{itemize}

\section*{Interpretation}
\begin{itemize}[leftmargin=1.5em]
\item The earlier Slack snippets were regression-only early checkpoints. The full locked run says the binary head is the more convincing part of this pair.
\item ``Ensemble beats last layer'' is too coarse:
  \begin{itemize}[leftmargin=1.5em]
  \item for regression, that does not imply a metadata win;
  \item for binary, the gain is real but still prevalence-sensitive on the rare-event datasets.
  \end{itemize}
\item The regression result should now be stated narrowly: \texttt{mean\_relative\_length} is still the stronger activation regression target relative to \texttt{last\_layer}, but this full-train pass does not show robust lift over the train-fit prompt-length control across the five-dataset suite.
\item The binary result is cleaner: \texttt{majority\_s\_0.5} is still not a pure loop label, but it is the stronger finished activation-lift surface on this run.
\end{itemize}
